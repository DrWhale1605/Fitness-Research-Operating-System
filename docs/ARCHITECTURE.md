# 1. Introduction

This document defines the software architecture of the Fitness Research Operating System (FROS).

Its purpose is to describe how the system is organized internally, how individual modules interact, and how information flows throughout the framework.

Unlike the System Requirements Specification (SRS), which defines *what* the system shall accomplish, this document defines *how* those requirements will be implemented through a modular software architecture.

The architecture has been designed to support incremental development, beginning with a local single-user Minimum Viable Product (MVP) and gradually evolving into the complete long-term framework defined by the project vision.

The architecture emphasizes:

- Modularity
- Scientific transparency
- Maintainability
- Extensibility
- Evidence-driven reasoning
- Clear separation of responsibilities

Every architectural decision should prioritize long-term maintainability over short-term implementation convenience.
